가중치 조정 → 순위 재계산. 실데이터: KTO 데이터랩 방문·소비 + 소노 숙세페 실적.

막대 = 콘텐츠 우선순위 점수(0~100). 가중합이 높을수록 먼저 제작.
X=월평균 방문(만) · Y=전환효율(매출÷방문) · 버블=비수기 낙폭. 우하단(방문 많은데 저전환)이 콘텐츠로 끌어올릴 1순위.
순위 · 시군구대표 시설방문(만)비수기낙폭전환효율외지인%방문 YoY우선순위점수

· 업로드(반자동): 승인 시 Instagram Graph API media→media_publish로 예약/게시. 키·토큰은 GitHub Secrets/서버에만 보관(대시보드·코드에 노출 금지)
· 네트워크 주의: 공공데이터·인스타 API는 인터넷 되는 실행부(GitHub Actions/서버)에서 호출. (현재 편집 클라우드·로컬 브리지는 외부 API 차단)
